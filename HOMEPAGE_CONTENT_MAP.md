# HOMEPAGE_CONTENT_MAP.md

## Purpose
Define the homepage content structure for Procitec.io.

## Homepage Objective
The homepage should help a busy industrial buyer quickly understand:
- what Procitec.io is
- what applications are available
- why it matters
- what action to take next

## Section Order

### 1. Header
- logo
- primary navigation
- prominent CTA

### 2. Hero
**Goal:** immediate clarity

Suggested messaging structure:
- Eyebrow: Procitec Applications
- Headline: Industrial applications built for reliability, visibility, and scale
- Supporting text: short product-platform explanation
- CTAs:
  - Open EAM
  - View Applications

### 3. Applications Overview
Cards:
- EAM
- EMS (coming soon)

Each card should include:
- short summary
- core benefit
- CTA

### 4. Problem → Solution Block
Themes:
- asset reliability
- maintenance visibility
- energy intelligence

### 5. How It Works
Simple 3-4 step explanation:
1. Connect data
2. Gain visibility
3. Manage action
4. Improve outcomes

### 6. Credibility Section
Can include:
- industries served
- multi-site suitability
- secure multi-tenant readiness
- industrial operating context

### 7. CTA Section
Direct user to:
- Open EAM
- Book Demo
- Talk to Expert

### 8. Footer
- product links
- support links
- corporate link to procitec.in
- contact

## Copy Rules
- keep short
- keep scannable
- avoid broad company history on homepage
- keep users moving toward product discovery
