# MOBILE_HOMEPAGE_LAYOUT.md

## Purpose
Define a mobile-first homepage layout for Procitec.io that Codex can implement directly.

Goal:
- Fast understanding
- Clear structure
- High conversion
- Minimal friction

---

## Layout Overview (Top to Bottom)

### 1. Header (Sticky)

- Logo (left)
- Hamburger menu (right)

Rules:
- Keep height small
- Ensure easy thumb reach

---

### 2. Hero Section (Above the Fold)

Content:
- Tagline (small text)
- Headline (2 lines max)
- Subtext (1–2 lines)
- Primary CTA
- Secondary CTA

Example:

Procitec Applications  
Industrial software for asset and energy intelligence  

[ Open EAM ]  
[ View Applications ]

Rules:
- No clutter
- Immediate clarity
- CTA visible without scrolling

---

### 3. Applications Section

Structure:
- Vertical cards

Each card:
- Title (EAM / EMS)
- Short description
- Key benefit
- CTA button

Rules:
- One card per row
- Large touch targets

---

### 4. Problem → Solution Section

Structure:

Problem:
Short, relatable issue

Solution:
How Procitec solves it

Outcome:
Clear benefit

Rules:
- Keep text short
- Use spacing

---

### 5. How It Works

Steps (vertical list):

1. Connect assets  
2. Monitor data  
3. Analyze insights  
4. Take action  

Rules:
- Use simple icons if needed
- Keep steps concise

---

### 6. Credibility Section

Include:
- Industries served
- Use cases
- Key strengths

Rules:
- Bullet points preferred
- Avoid long paragraphs

---

### 7. CTA Section

Strong action:

[ Open EAM ]  
[ Book Demo ]

Rules:
- Full-width buttons
- High contrast

---

### 8. Footer

Include:
- Navigation links
- Contact info
- Minimal text

---

## Mobile Design Rules

### Layout
- Single column
- Clear spacing between sections

### Typography
- Large readable text
- Strong hierarchy

### Buttons
- Full width or large size
- Minimum height ~44px

### Spacing
- Use padding generously
- Avoid cramped layouts

---

## Performance Rules

- Fast loading
- Optimize images
- Avoid heavy JS

---

## Codex Implementation Rules

When building:

- Use Tailwind responsive classes
- Start with mobile layout first
- Then enhance for desktop

Do NOT:
- Design desktop first
- Add unnecessary UI elements

---

## Final Principle

Mobile homepage must feel like:

- fast
- clear
- actionable

User should:
- understand instantly
- take action quickly
