# MOBILE_EXPERIENCE_STRATEGY.md

## Purpose
Ensure Procitec.io delivers a high-impact, fast, and engaging experience on mobile devices.

Target users are busy industrial professionals who:
- scan quickly
- make decisions fast
- prefer clarity over exploration

---

## Core Principles

### 1. Mobile-first thinking
Design for mobile first, then scale up.

### 2. 5-second rule
User must understand:
- what this is
- how it helps
- what to do next

within 5 seconds.

### 3. Thumb-friendly design
- Large tap targets
- Easy navigation
- No tiny links

---

## Homepage (Mobile Priority)

### Hero Section
- Clear headline (2 lines max)
- Short subtext
- 1 primary CTA
- 1 secondary CTA

### Applications Section
- Stack cards vertically
- Keep content short
- Strong CTA buttons

### Navigation
- Simple hamburger menu
- Max 5–6 items

---

## Layout Rules

- Single column layout
- Adequate spacing (no clutter)
- Sections clearly separated

---

## Performance (CRITICAL)

- Fast loading (<2–3 sec)
- Optimize images
- Avoid heavy animations

Slow mobile = lost customer

---

## Content Rules

- Short sentences
- Bullet points
- Avoid long paragraphs

---

## Interaction Design

### Include
- Smooth scrolling
- Subtle transitions
- Clear button feedback

### Avoid
- Complex animations
- Heavy JS interactions

---

## CTA Strategy

- Place CTA above the fold
- Repeat CTA every 2–3 sections

Examples:
- Open EAM
- Book Demo
- Talk to Expert

---

## Codex Implementation Rules

When generating mobile UI:

- Use responsive Tailwind classes
- Prioritize mobile layout first
- Test small screen behavior
- Ensure tap targets are usable

---

## Testing Checklist

- Test on real phone
- Check load speed
- Check readability
- Check navigation ease
- Check CTA visibility

---

## Final Principle

Mobile success =

Speed + Clarity + Simplicity

If user can understand and act instantly,
engagement will follow.
